## 💡 项目简介

在以大语言模型为核心的自主智能体开发中，传统的“裸奔运行”智能体正面临着严峻的生存挑战：接口调用频繁爆仓所带来的极高资金开销、沙箱环境中执行恶意代码导致的越权安全风险，以及逻辑或代码错误引发死循环而产生的接口请求风暴。这些痛点严重阻碍了智能体从原型验证走向工业级落地的进程。

作为 OpenClaw 等高权限自主智能体生态中不可或缺的拼图，**Daming OS (大明天子 OS)** 应运而生。它致力于为高权限自主智能体提供工业级的防弹底座与自演化记忆外脑。Daming OS 提炼自高度复杂的企业级生产实践，将底层的防爆仓记忆引擎与防越权成长沙箱完全抽离解耦，打造为一个轻量且即插即用的软件库，为智能体的安全运行与持续演化保驾护航。

### ✨ 核心特性

#### 一、 记忆系统核心特性
* **按需检索与多维搜索**：响应全局配置，仅在系统报错、遭遇瓶颈或求助时唤醒记忆库，极致优化上下文开销。深度整合密集向量与全文稀疏检索，实现基于语义相似度与文本匹配的多维极速定位。
* **物理排序与长度截断**：多路检索结果融合排序，并在返回前进行物理长度截断，强制进行字符切片以过滤低价值长文，从物理层面上杜绝因单条记忆过长导致的直接爆仓。
* **记忆去嵌套与防爆仓网关**：内置信息密度筛选算法，在存入记忆库前自动清洗输入中的历史提示词标签，避免智能体陷入循环记忆的逻辑死结；在额度超限时自适应精简历史，并将冗余物理固化旁路归档。
* **两级高并发语义缓存**：基于文件锁实现微秒级热数据响应，避免频繁读取数据库。
* **原子态会话记录机**：通过文件级原子替换与防撕裂写入，确保当前会话的存储安全无竞态。

#### 二、 成长系统核心特性
* **异常行为捕获器**：实时监听运行报错日志，利用指数衰退滑动窗口计算成长值积分，积满触发能力演化。
* **多智能体演化辩论机制**：基于红方挑刺、蓝方修复、白方总结的博弈架构，自动提取高质量代码修复补丁与最佳实践.
* **静态安检门与隔离沙箱**：在隔离沙箱内运行编译级安全检测、静态分析与冒烟测试，强制拦截并封锁高危导入与文件系统反射修改，严防越权与提权，确保代码无毒。
* **人类在环授权与毫秒级部署**：生成审核卡片交由管理员确认，采用文件级替换实现无感热插拔式自愈与毫秒级原子部署，同时实时保留物理冷备，支持一键安全回滚。

#### 三、 两大系统的闭环交互机制
* **异常沉淀为记忆**：成长系统拦截智能体运行中的所有报错，转化为日志作为负反馈记忆沉淀至底层数据库。
* **召回历史教训**：当成长系统触发代码辩论演化时，记忆系统自动召回历史相似报错与避坑指南，避免重复犯错。
* **部署后刷新缓存**：成长系统完成代码部署后发布完成事件，记忆系统接收并清空缓存，使智能体下轮即刻应用新行为规范。

## 🚀 一键极速安装

在您的终端中执行以下一键安装命令，即可直接从远程 GitHub 仓库拉取并配置包及其全部依赖，无需繁琐的手动克隆和配置：
```bash
pip install git+https://github.com/dylanma8232-art/Daming-OS.git
```

### 脚手架一键生成工作区
安装完成后，在您自己的智能体项目根目录下，运行命令行工具一键生成配置骨架：
```bash
daming-os init --dir ./my-agent-workspace
```
这将在目标目录下自动生成 `AGENTS.md`（指令规范）、`USER.md`（管理员授权配置）、`.env` 配置文件等。

## 🔐 极限安全声明
Daming OS 秉持**极端防御主义理念**，在安全门控中默认封锁一切高级提权和文件系统反射修改。如开发者的智能体确实需要执行高权限系统操作，请在配置中进行策略授权，或自行微调安检名单以保证合规。
