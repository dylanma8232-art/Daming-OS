# 参与贡献指南 (Contributing to Daming OS)

感谢您对 Daming OS 的关注！作为一款工业级的智能体防弹底座，我们欢迎极客和开发者们提交 Bug 和需求。

## 🐞 提交 Bug (Bug Reports)
如果您发现了系统漏洞或报错：
1. 请先在 Issues 列表里搜索是否有人已经提过。
2. 点击 `New Issue`，选择 `Bug Report` 模板。
3. **必须提供复现步骤和完整的错误日志**，否则无法处理。

## 💡 提交需求 (Feature Requests)
如果您对大明 OS（特别是大明记忆系统或成长沙箱）有颠覆性的架构建议：
1. 点击 `New Issue`，选择 `Feature Request` 模板。
2. 讲清楚核心的“痛点”，而不是单纯要求加一个按钮。
3. 只有符合《大明天子白皮书》核心理念的需求才会被合并。

## 🛠 提交代码 (Pull Requests)
1. Fork 本仓库。
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)。
3. 提交您的修改 (`git commit -m 'Add some AmazingFeature'`)。
4. 推送到分支 (`git push origin feature/AmazingFeature`)。
5. 发起一个 Pull Request。

**⚠️ 安全红线要求**：
所有提交的代码，不得绕过 `Growth System` 中的 AST 安检门，不得引入任何未经授权的隐藏遥测机制。
